## Topic 101：系統架構

### 101.1 判斷與設定硬體設定（權重：2）

#### 考試重點
- 了解 BIOS/UEFI 設定如何影響作業系統與硬體的互動
- 使用 Linux 工具查詢硬體資訊
- 識別永久與可移除儲存裝置

#### 查詢硬體資訊

```bash
lspci           # 列出所有 PCI 裝置（顯示卡、網卡、控制器等）
lspci -v        # 詳細資訊
lspci -k        # 顯示每個裝置使用的核心驅動程式

lsusb           # 列出所有 USB 裝置
lsusb -v        # 詳細資訊
lsusb -t        # 樹狀結構顯示

lsmod           # 列出目前載入的核心模組
modprobe <mod>  # 載入模組（含相依性）
modprobe -r <mod>  # 卸載模組
insmod <mod>    # 直接插入模組（不處理相依性）
rmmod <mod>     # 移除模組
modinfo <mod>   # 查看模組資訊
```

#### 重要偽檔案系統（虛擬檔案）

| 路徑 | 說明 |
|------|------|
| `/proc/` | 行程與核心資訊（在記憶體中，非真實檔案）|
| `/proc/cpuinfo` | CPU 資訊 |
| `/proc/meminfo` | 記憶體資訊 |
| `/proc/interrupts` | IRQ 中斷資訊 |
| `/proc/ioports` | I/O 連接埠 |
| `/proc/bus/usb/` | USB 裝置 |
| `/sys/` | 核心裝置資訊（sysfs）|
| `/dev/` | 裝置檔案 |

#### 儲存裝置命名規則

| 裝置 | 命名 | 說明 |
|------|------|------|
| SATA/SCSI 硬碟 | `/dev/sda`, `/dev/sdb` | 第一顆、第二顆 |
| NVMe SSD | `/dev/nvme0n1` | NVMe 第一顆 |
| 光碟機 | `/dev/sr0` | SATA/USB 光碟 |
| 分割區 | `/dev/sda1`, `/dev/sda2` | 硬碟的分割區 |

### 101.2 開機系統（權重：3）

#### Linux 開機流程

```
BIOS/UEFI
    ↓
Boot Loader（GRUB 2）
    ↓
Linux Kernel（vmlinuz）
    ↓
initramfs（暫時根檔案系統）
    ↓
init / systemd（PID 1）
    ↓
使用者空間服務啟動完成
```

#### BIOS vs UEFI

| | BIOS | UEFI |
|-|------|------|
| 分割表 | MBR | GPT（支援 >2TB）|
| 開機分割區 | 無 | ESP（EFI System Partition）|
| 安全啟動 | 無 | 支援 Secure Boot |
| 介面 | 文字 | 圖形/文字 |

#### 核心啟動參數（重要考點）

常見的核心參數可在 GRUB 開機選單中加入：

| 參數 | 說明 |
|------|------|
| `root=/dev/sda1` | 指定根檔案系統 |
| `ro` / `rw` | 以唯讀/可寫掛載根 |
| `quiet` | 抑制開機訊息 |
| `single` / `1` | 進入單人模式（救援）|
| `init=/bin/bash` | 指定 init 程式（緊急救援）|
| `mem=512M` | 限制可用記憶體 |

#### 查看開機訊息

```bash
dmesg                   # 顯示核心環形緩衝區訊息（開機訊息）
dmesg | grep -i error   # 篩選錯誤訊息
dmesg | grep -i usb     # 篩選 USB 相關訊息
dmesg -T                # 顯示可讀時間戳記

journalctl -b           # 本次開機的所有 systemd 日誌
journalctl -b -1        # 上一次開機的日誌
journalctl -k           # 只看核心訊息（等同 dmesg）
```

### 101.3 更改執行層級/開機目標與關機或重啟（權重：3）

#### SysVinit 執行層級（Runlevel）

| Runlevel | 說明 |
|---------|------|
| 0 | 關機 |
| 1 | 單人模式（維護）|
| 2 | 多人模式（無網路，Debian 慣例）|
| 3 | 多人模式（命令列，RedHat 慣例）|
| 4 | 未使用（自訂）|
| 5 | 多人模式（含圖形介面）|
| 6 | 重新開機 |

```bash
# SysVinit 相關指令
runlevel        # 查看目前執行層級（輸出：前一個 目前的）
telinit 3       # 切換到 runlevel 3
init 0          # 關機
init 6          # 重啟

# 設定預設 runlevel → /etc/inittab
# id:5:initdefault:
```

#### systemd Target（現代系統）

| Target | 對應 Runlevel | 說明 |
|--------|-------------|------|
| `poweroff.target` | 0 | 關機 |
| `rescue.target` | 1 | 救援模式 |
| `multi-user.target` | 3 | 多人命令列 |
| `graphical.target` | 5 | 圖形介面 |
| `reboot.target` | 6 | 重啟 |
| `emergency.target` | — | 緊急模式 |

```bash
# systemd 相關指令
systemctl get-default                    # 查看預設 target
systemctl set-default graphical.target   # 設定預設 target
systemctl isolate multi-user.target      # 立即切換 target

systemctl list-units --type=target       # 列出所有 target

# 服務管理
systemctl start/stop/restart sshd
systemctl enable/disable sshd           # 設定開機自啟
systemctl status sshd
```

#### 關機與重啟指令

```bash
shutdown -h now         # 立即關機（-h = halt）
shutdown -h +10         # 10分鐘後關機
shutdown -h 22:30       # 指定時間關機
shutdown -r now         # 立即重啟（-r = reboot）
shutdown -c             # 取消排程中的關機

poweroff                # 關機
reboot                  # 重啟
halt                    # 停止系統

wall "系統將在10分後維護"  # 廣播訊息給所有已登入使用者
```

## Topic 102：Linux 安裝與套件管理

### 102.1 設計硬碟分割配置（權重：2）

#### 重要掛載點規劃

| 目錄 | 建議獨立分割 | 原因 |
|------|------------|------|
| `/` | 必要 | 根目錄 |
| `/boot` | 建議 | 存放開機核心，避免 LVM 問題 |
| `/home` | 建議 | 用戶資料，避免系統分割區被填滿 |
| `/var` | 伺服器建議 | 日誌/郵件/資料庫，成長快速 |
| `/tmp` | 可選 | 暫存檔，可設 noexec |
| `/usr` | 可選 | 程式檔案 |
| `swap` | 必要 | 虛擬記憶體 |

#### Swap 空間規劃

| RAM 大小 | 建議 Swap |
|---------|---------|
| ≤ 2 GB | RAM × 2 |
| 2–8 GB | RAM × 1 |
| 8–64 GB | 4–8 GB |
| > 64 GB | 視需求（8 GB+）|

#### LVM（邏輯磁碟區管理）概念

```
實體磁碟 (/dev/sda, /dev/sdb)
    ↓ pvcreate
PV（Physical Volume，實體磁碟區）
    ↓ vgcreate
VG（Volume Group，磁碟區群組）
    ↓ lvcreate
LV（Logical Volume，邏輯磁碟區）→ 可動態調整大小
```

**LVM 優點：** 可跨多顆硬碟、線上擴充空間、快照功能。

### 102.2 安裝開機管理程式（權重：2）

#### GRUB 2（目前主流）

**重要檔案：**
```
/boot/grub/grub.cfg           # 主設定檔（自動生成，不要手動改）
/etc/grub.d/                  # 設定腳本目錄
/etc/default/grub             # GRUB 預設參數設定
```

**修改設定後必須執行：**
```bash
update-grub                   # Debian/Ubuntu
grub-mkconfig -o /boot/grub/grub.cfg  # 通用（RedHat 等）
```

**安裝 GRUB：**
```bash
grub-install /dev/sda         # 安裝到 MBR（BIOS 系統）
grub-install --target=x86_64-efi /dev/sda  # 安裝到 EFI
```

**`/etc/default/grub` 常見設定：**
```bash
GRUB_DEFAULT=0                # 預設選單項目
GRUB_TIMEOUT=5                # 等待秒數
GRUB_CMDLINE_LINUX="quiet splash"  # 核心參數
GRUB_DISABLE_RECOVERY="true"  # 隱藏救援選單
```

#### GRUB Legacy vs GRUB 2

| | GRUB Legacy | GRUB 2 |
|-|------------|--------|
| 設定檔 | `/boot/grub/menu.lst` 或 `grub.conf` | `/boot/grub/grub.cfg` |
| 磁碟計數 | 從 0 開始（hd0）| 同左 |
| 分割區計數 | 從 0 開始 | 從 1 開始 |
| 設定方式 | 直接編輯設定檔 | 透過腳本自動生成 |

### 102.3 管理共享函式庫（權重：1）

#### 概念

- **靜態函式庫（Static Library）**：`.a` 副檔名，編譯時整合進執行檔，體積大但獨立
- **共享函式庫（Shared Library）**：`.so` 副檔名，執行時才載入，多程式共用節省空間

**命名規則：**
```
libname.so.主版本號
libcurl.so.4   ← soname（符號連結指向）
libcurl.so.4.7.0  ← 實際檔案
libcurl.so     ← 開發用符號連結
```

#### 函式庫路徑設定

```bash
# 預設搜尋路徑
/lib/
/usr/lib/
/usr/local/lib/

# 設定額外路徑（兩種方式）
# 方法一：編輯設定檔
echo "/opt/mylib" >> /etc/ld.so.conf.d/mylib.conf
ldconfig                    # 必須執行以更新快取

# 方法二：環境變數（暫時）
export LD_LIBRARY_PATH=/opt/mylib:$LD_LIBRARY_PATH

# 查詢執行檔的函式庫相依
ldd /usr/bin/ls             # 列出 ls 的所有相依函式庫
```

### 102.4 使用 Debian 套件管理（權重：3）

#### dpkg — 底層套件工具

```bash
# 安裝
dpkg -i package.deb         # 安裝 .deb 套件
dpkg -r package             # 移除套件（保留設定檔）
dpkg -P package             # 完全移除套件（含設定檔）

# 查詢
dpkg -l                     # 列出所有已安裝套件
dpkg -l | grep nginx        # 搜尋特定套件
dpkg -L package             # 列出套件安裝了哪些檔案
dpkg -S /usr/bin/vim        # 查詢檔案屬於哪個套件
dpkg -I package.deb         # 查看 .deb 套件資訊（未安裝）
dpkg --get-selections       # 列出所有套件與狀態

# 設定
dpkg-reconfigure package    # 重新設定已安裝套件
```

#### apt — 高層套件管理工具（含相依性解析）

```bash
# 套件庫更新
apt update                  # 更新套件清單（必須先執行）
apt upgrade                 # 升級所有套件
apt full-upgrade            # 升級（可移除衝突套件）

# 安裝與移除
apt install nginx           # 安裝套件
apt install nginx=1.18.0    # 安裝指定版本
apt remove nginx            # 移除套件（保留設定）
apt purge nginx             # 完全移除（含設定）
apt autoremove              # 移除不需要的相依套件

# 搜尋與資訊
apt search nginx            # 搜尋套件
apt show nginx              # 顯示套件詳細資訊
apt list --installed        # 列出已安裝套件

# 舊式指令（apt-get / apt-cache）
apt-get update
apt-get install nginx
apt-cache search nginx
apt-cache show nginx

# 查詢檔案屬於哪個套件（需安裝 apt-file）
apt-file update
apt-file search /usr/bin/vim
apt-file list nginx         # 列出套件包含的檔案
```

#### 套件庫設定

```bash
# 主要設定檔
/etc/apt/sources.list
/etc/apt/sources.list.d/   # 附加套件庫目錄

# sources.list 格式
# deb [選項] URL 發行版 元件
deb http://archive.ubuntu.com/ubuntu focal main restricted universe multiverse
deb-src http://archive.ubuntu.com/ubuntu focal main   # 原始碼套件庫
```

### 102.5 使用 RPM 與 YUM 套件管理（權重：3）

#### rpm — 底層套件工具（RedHat 系）

```bash
# 安裝與移除
rpm -i package.rpm          # 安裝套件
rpm -iv package.rpm         # 安裝（詳細輸出）
rpm -ivh package.rpm        # 安裝（顯示進度條）
rpm -U package.rpm          # 升級（若未安裝則安裝）
rpm -e package              # 移除套件

# 查詢（-q 系列）
rpm -qa                     # 列出所有已安裝套件
rpm -qi package             # 查看套件資訊
rpm -ql package             # 列出套件安裝的所有檔案
rpm -qf /usr/bin/vim        # 查詢檔案屬於哪個套件
rpm -qip package.rpm        # 查看未安裝的 .rpm 資訊
rpm -qlp package.rpm        # 列出未安裝的 .rpm 檔案清單

# 驗證
rpm -V package              # 驗證套件完整性
rpm --checksig package.rpm  # 驗證 GPG 簽章
```

#### yum — 高層套件管理（CentOS/RHEL）

```bash
# 更新與安裝
yum check-update            # 檢查可更新套件
yum update                  # 更新所有套件
yum update package          # 更新特定套件
yum install package         # 安裝套件
yum remove package          # 移除套件
yum reinstall package       # 重新安裝

# 搜尋與資訊
yum search keyword          # 搜尋套件
yum info package            # 顯示套件資訊
yum list installed          # 列出已安裝套件
yum list available          # 列出可安裝套件
yum provides /usr/bin/vim   # 查詢哪個套件提供此檔案

# 套件庫管理
yum repolist                # 列出啟用中的套件庫
yum repolist all            # 列出所有套件庫
yum-config-manager --add-repo URL    # 新增套件庫
yum-config-manager --enable repo-id  # 啟用套件庫
yum-config-manager --disable repo-id # 停用套件庫
yum clean all               # 清理快取

# 套件庫設定目錄
/etc/yum.repos.d/           # .repo 設定檔
/etc/yum.conf               # 主設定檔
```

#### dnf — yum 的現代替代（Fedora/RHEL 8+）

```bash
dnf install package         # 與 yum 語法幾乎相同
dnf remove package
dnf search keyword
dnf update
dnf list installed
dnf provides /usr/bin/vim
dnf repolist
dnf clean all
```

#### zypper — openSUSE 套件管理

```bash
zypper refresh              # 更新套件庫清單
zypper update               # 更新套件
zypper install package      # 安裝
zypper remove package       # 移除
zypper search keyword       # 搜尋
zypper info package         # 套件資訊
zypper repos                # 列出套件庫
zypper addrepo URL alias    # 新增套件庫
```

#### 各發行版套件管理對照表

| 功能 | Debian/Ubuntu | RedHat/CentOS | openSUSE |
|------|-------------|--------------|---------|
| 套件格式 | `.deb` | `.rpm` | `.rpm` |
| 底層工具 | `dpkg` | `rpm` | `rpm` |
| 高層工具 | `apt` | `yum` / `dnf` | `zypper` |
| 套件庫設定 | `/etc/apt/sources.list` | `/etc/yum.repos.d/` | `/etc/zypp/repos.d/` |

### 102.6 Linux 作為虛擬化訪客（權重：1）

#### 虛擬化類型

| 類型 | 說明 | 範例 |
|------|------|------|
| **完全虛擬化** | 硬體完全模擬 | VirtualBox、VMware |
| **半虛擬化** | 訪客知道自己在虛擬環境 | Xen（paravirt 模式）|

| **容器** | 共享主機核心，隔離行程 | Docker、LXC、Podman |

#### 雲端部署相關

```bash
# 查看系統的 DBus UUID（VM 複製後需更新）

# cloud-init：雲端 VM 初始化工具
# 設定檔：/etc/cloud/cloud.cfg

# VM 範本複製後應做的事
# 1. 更新機器 ID
rm /etc/machine-id && systemd-machine-id-setup

rm /etc/ssh/ssh_host_* && dpkg-reconfigure openssh-server
```


#### 指令歷史

```bash
history             # 顯示指令歷史

# 相關環境變數

HISTFILESIZE=2000   # 歷史檔案的最大行數
HISTFILE=~/.bash_history  # 歷史檔案位置
```

#### 引號與跳脫字元

| 符號 | 說明 |
|------|------|
| `"..."` | 雙引號：允許變數展開，但不允許萬用字元 |
| `'...'` | 單引號：完全保留字面值，無任何展開 |
| `\` | 跳脫下一個字元（保留字面值）|
| `` ` `` | 反引號：指令替換（已不推薦，用 `$()` 代替）|

```bash
echo "Home is $HOME"    # 輸出：Home is /home/user
echo 'Home is $HOME'    # 輸出：Home is $HOME
echo "Cost: \$5"        # 輸出：Cost: $5
```

### 103.2 使用過濾器處理文字串流（權重：3）

#### 重新導向與管道

```bash
# 輸出重新導向
command > file      # 覆寫輸出至檔案
command >> file     # 附加輸出至檔案
command 2> file     # 將標準錯誤導向檔案
command 2>&1        # 將 stderr 合併到 stdout
command &> file     # stdout + stderr 都導向檔案
command > file 2>&1 # 同上（舊式寫法）

# 輸入重新導向
command < file      # 從檔案讀取輸入

# 管道
command1 | command2 # 將 command1 的輸出傳給 command2

# 丟棄輸出
```

### 103.6 修改行程執行優先順序（權重：2）

#### Nice 值概念

- Nice 值範圍：**-20（最高優先）到 +19（最低優先）**
- 一般使用者只能調高 nice 值（降低優先權）
- root 可以調低 nice 值（提高優先權）
- **預設 nice 值為 0**

```bash
# 啟動時指定優先順序
nice -n 10 command      # 以 nice 值 10 執行（較低優先）
nice -n -5 command      # 以 nice 值 -5 執行（需root，較高優先）

# 修改已執行行程的優先順序
renice 10 -p PID        # 設定 PID 的 nice 值為 10
renice 10 -u username   # 設定某使用者所有行程

# 查看優先順序
ps -el | head           # 查看 PRI 和 NI 欄位
top                     # NI 欄位顯示 nice 值
```

**ps 輸出的 PR（Priority）與 NI（Nice）：**
- `PR = 20 + NI`（通常）
- `PR = rt` 表示即時行程（realtime）

### 104.1 建立分割區與檔案系統（權重：2）

#### MBR vs GPT

| | MBR | GPT |
|-|-----|-----|
| 最大磁碟 | 2 TB | 9.4 ZB |
| 最多分割區 | 4個主分割（或3主+1延伸）| 128個（預設）|
| 開機韌體 | BIOS | UEFI（建議）|
| 備份分割表 | 無 | 磁碟結尾有備份 |

#### 分割工具

```bash
# fdisk（MBR 分割，互動式）
fdisk /dev/sda      # 進入互動模式
# 常用指令：
# m = 顯示說明
# p = 顯示分割表
# n = 新增分割區
# d = 刪除分割區
# t = 更改分割區類型
# w = 寫入並離開
# q = 離開不儲存

# gdisk（GPT 分割）
gdisk /dev/sda      # 語法與 fdisk 類似

# parted（支援 MBR 和 GPT，可腳本化）
parted /dev/sda print       # 顯示分割資訊
parted /dev/sda mklabel gpt # 建立 GPT 分割表
parted /dev/sda mkpart primary ext4 1MiB 100GiB  # 建立分割
parted /dev/sda rm 1        # 刪除分割區 1
parted /dev/sda resizepart 1 50GiB  # 調整大小
```

#### 建立檔案系統

```bash
mkfs.ext4 /dev/sda1         # 建立 ext4（最常用）
mkfs.ext3 /dev/sda1         # 建立 ext3
mkfs.ext2 /dev/sda1         # 建立 ext2
mkfs.xfs /dev/sda1          # 建立 XFS
mkfs.vfat /dev/sda1         # 建立 FAT32（USB、EFI 分割區）
mkfs.exfat /dev/sda1        # 建立 exFAT
mkfs -t ext4 /dev/sda1      # 另一種語法

# 建立 Swap 分割區
mkswap /dev/sda2            # 格式化為 swap
swapon /dev/sda2            # 啟用 swap
swapoff /dev/sda2           # 停用 swap
swapon -s                   # 查看目前 swap
free -h                     # 確認 swap 使用情況
```

| 檔案系統 | 特色 | 使用場景 |

### 104.3 控制檔案系統掛載與卸載（權重：3）

#### 手動掛載與卸載

```bash
# 掛載
mount /dev/sda1 /mnt/data           # 掛載到指定點
mount -t ext4 /dev/sda1 /mnt/data   # 指定檔案系統類型
mount -o ro /dev/sda1 /mnt          # 以唯讀掛載
mount -o remount,rw /                # 重新掛載（改為可寫）

mount                               # 顯示所有已掛載的檔案系統
mount | grep sda1

# 卸載
umount /mnt/data                    # 按掛載點卸載
umount /dev/sda1                    # 按裝置卸載
umount -f /mnt/data                 # 強制卸載（謹慎使用）

lsof /mnt/data                      # 查看是否有行程占用（若無法卸載時用）

# 查看裝置資訊
lsblk                               # 以樹狀顯示所有區塊裝置
lsblk -f                            # 含檔案系統類型和 UUID
blkid                               # 顯示所有裝置的 UUID 和類型
blkid /dev/sda1
```

#### 自動掛載設定 /etc/fstab

```
# 裝置          掛載點    檔案系統  選項         dump  pass
/dev/sda1       /         ext4      defaults      1     1
/dev/sda2       /boot     ext4      defaults      1     2
/dev/sdb1       /data     xfs       defaults      0     0
UUID=abc-123    /home     ext4      defaults      0     2
LABEL=mydata    /backup   ext4      defaults      0     0
/dev/sdc1       swap      swap      defaults      0     0
tmpfs           /tmp      tmpfs     defaults      0     0
```

**選項說明：**

| 選項 | 說明 |
|------|------|
| `defaults` | rw,suid,dev,exec,auto,nouser,async |
| `ro` / `rw` | 唯讀 / 可讀寫 |
| `noexec` | 禁止執行程式 |
| `nosuid` | 禁用 SUID/SGID |
| `noatime` | 不更新存取時間（效能優化）|
| `auto` / `noauto` | 是否在 `mount -a` 時自動掛載 |
| `user` / `nouser` | 是否允許一般使用者掛載 |

**dump 欄位：** 0=不備份，1=備份
**pass 欄位：** 0=不檢查，1=先檢查（root），2=後檢查（其他）

```bash
mount -a    # 掛載 /etc/fstab 中所有應自動掛載的項目（測試用）
```

#### 使用 UUID 和標籤

```bash
blkid /dev/sda1                     # 查看 UUID
lsblk -f                            # 查看所有裝置 UUID

# 在 fstab 中使用 UUID（推薦，裝置名稱可能改變）
UUID=xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx  /data  ext4  defaults  0  2
```

### 104.5 管理檔案權限與擁有權（權重：3）

#### 權限結構

```
-rwxr-xr-x  1  user  group  1234  Jan 1 12:00  file
│└──┘└──┘└──┘
│ 擁有者 群組 其他人
│
└ 檔案類型：- 一般檔案，d 目錄，l 符號連結，b 區塊裝置，c 字元裝置
```

| 權限 | 對檔案 | 對目錄 | 數字 |
|------|--------|--------|------|
| `r`（讀取）| 可讀取內容 | 可列出內容（ls）| 4 |
| `w`（寫入）| 可修改內容 | 可建立/刪除檔案 | 2 |
| `x`（執行）| 可執行 | 可進入目錄（cd）| 1 |

#### chmod — 更改權限

```bash
# 數字模式（八進位）
chmod 755 file      # rwxr-xr-x（owner=7, group=5, others=5）
chmod 644 file      # rw-r--r--（常見檔案權限）
chmod 600 file      # rw-------（私密檔案）
chmod 777 file      # rwxrwxrwx（全開放，慎用）
chmod -R 755 dir/   # 遞迴設定目錄

# 符號模式
chmod u+x file      # 擁有者加執行權
chmod g-w file      # 群組移除寫入權
chmod o=r file      # 其他人設為只讀
chmod a+x file      # 所有人加執行權（a = all）
chmod u+x,g-w file  # 多個設定
```

**常用權限數字記憶：**

| 數字 | 權限 | 適用場景 |
|------|------|---------|
| 644 | rw-r--r-- | 一般檔案 |
| 755 | rwxr-xr-x | 程式/目錄 |
| 600 | rw------- | SSH 私鑰 |
| 700 | rwx------ | 個人腳本 |
| 664 | rw-rw-r-- | 群組共同編輯 |

#### 特殊權限位元

| 權限 | 數字 | 對檔案 | 對目錄 |
|------|------|--------|--------|
| SUID | 4000 | 以擁有者身分執行 | 無效 |
| SGID | 2000 | 以群組身分執行 | 新檔繼承群組 |
| Sticky bit | 1000 | 無效 | 只有擁有者能刪自己的檔 |

```bash
chmod u+s file      # 設定 SUID
chmod g+s dir/      # 設定 SGID
chmod +t dir/       # 設定 Sticky bit
chmod 4755 file     # 數字方式設定 SUID

ls -l /usr/bin/passwd   # -rwsr-xr-x（s = SUID）
ls -ld /tmp             # drwxrwxrwt（t = sticky bit）
```

#### chown / chgrp — 更改擁有者

```bash
chown user file             # 更改擁有者
chown user:group file       # 更改擁有者和群組
chown :group file           # 只更改群組
chown -R user:group dir/    # 遞迴更改

chgrp group file            # 只更改群組
chgrp -R group dir/
```

#### umask — 預設權限遮罩

```bash
umask               # 查看目前 umask（通常是 022）
umask 002           # 設定 umask

# 計算方式：
# 檔案最大權限：666（沒有 x）
# 目錄最大權限：777
# umask 022：
#   檔案：666 - 022 = 644
#   目錄：777 - 022 = 755
# umask 002：
#   檔案：664
#   目錄：775
```

### 104.6 建立與更改硬連結與符號連結（權重：2）

#### 硬連結 vs 符號連結

| | 硬連結（Hard Link）| 符號連結（Symbolic Link）|
|-|-------------------|------------------------|
| 指向 | 相同 inode | 另一個路徑名稱 |
| 跨檔案系統 | ❌ 不支援 | ✅ 支援 |
| 目錄 | ❌ 不支援 | ✅ 支援 |
| 原檔刪除 | 連結仍有效 | 連結失效（dangling）|
| 識別 | 相同 inode 號碼 | `ls -l` 顯示 `->` |

```bash
ln file hardlink            # 建立硬連結
ln -s file symlink          # 建立符號連結（-s = symbolic）
ln -s /path/to/file link    # 絕對路徑符號連結
ln -s ../file link          # 相對路徑符號連結

ls -li file hardlink        # 查看 inode（相同則為硬連結）
ls -l symlink               # 查看符號連結指向

# 移除連結
rm hardlink                 # 刪除硬連結
rm symlink                  # 刪除符號連結（不影響原檔）
```
